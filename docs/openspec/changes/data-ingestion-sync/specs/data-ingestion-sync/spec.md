# 数据拉取与同步规范

## 新增需求

### 需求：显式同步是默认主链路

系统必须将显式触发的同步作为数据拉取的默认主链路。

#### 场景：按指定代码手动同步

- **给定** 用户希望刷新某只股票的本地数据
- **当** 用户以指定市场和股票代码发起同步时
- **那么** 系统显式启动一次同步任务
- **并且** 正常查询链路不会因为本地缺失而自动回源

### 需求：为辅助性定时同步保留扩展入口

系统必须为辅助性的定时同步保留扩展入口，但不得把它替代显式手动同步成为默认主链路。

#### 场景：接入定时触发器

- **给定** 系统后续需要为指定股票配置辅助性定时同步
- **当** 接入新的定时触发方式时
- **那么** 系统可以复用既有同步流程执行任务
- **并且** 显式手动同步仍然是默认主链路

### 需求：按优先顺序兜底数据源

系统必须按照预设优先级依次尝试可用数据源，并采用首个成功的标准化结果。

#### 场景：首选数据源失败

- **给定** 首选数据源不可用或返回无效数据
- **当** 同步流程继续执行时
- **那么** 系统按预设顺序尝试下一个可用数据源

### 需求：记录每次同步结果

系统必须持久化每次同步尝试的结果和关键信息。

#### 场景：记录一次失败同步

- **给定** 某次同步请求执行失败
- **当** 同步任务结束时
- **那么** 系统保存失败状态、使用数据源、开始时间、结束时间和失败摘要
