# Design

## 目标

为“当前筛选”和“历史明细”提供两套不同的数据承载方式，避免一张表同时承担两类相反的查询需求。

## 核心决策

### 1. 双表思路

财务数据至少拆成两层：

- 历史财务表：保存期间维度的历史结果
- 财务筛选快照表：保存当前可筛选的最新快照

### 2. 查询路径

- 多指标筛选优先查快照表
- 个股详情查看历史时，再查历史表

这样可以让组合筛选更稳定，也便于以后补索引。

### 3. 指标来源边界

正式财务结果优先来自结构化数据源。

当结构化数据拿不到时，可以把公告或年报 PDF 的解析结果作为人工辅助，但不自动覆盖正式快照。

## 依赖边界

这个能力域依赖：

- `stock-registry`：统一股票身份
- `data-ingestion-sync`：拿到外部原始财务数据

## 风险与约束

- 不同数据源的财务口径可能不完全一致，需要在适配层做标准化
- 多指标查询会不断扩展，表结构和索引必须优先面向筛选场景设计
