# 财务筛选规范

## 新增需求

### 需求：维护独立的财务历史数据

系统必须将历史财务记录与筛选快照分开维护。

#### 场景：保存按报告期归档的历史财务

- **给定** 某只股票已经获取到结构化财务数据
- **当** 系统保存财务结果时
- **那么** 系统按报告期写入财务历史存储

### 需求：维护用于筛选的最新快照

系统必须维护一份用于快速筛选的当前财务快照。

#### 场景：刷新最新筛选值

- **给定** 某只股票获取到了新的结构化财务数据
- **当** 系统更新筛选数据集时
- **那么** 系统刷新该股票用于筛选的最新快照

### 需求：支持多指标组合筛选

系统必须支持基于当前快照进行多指标组合筛选。

#### 场景：按多个财务条件筛选股票

- **给定** 用户选择了多个财务筛选条件
- **当** 筛选查询执行时
- **那么** 系统基于最新财务快照返回符合条件的股票结果

### 需求：提供个股财务历史查询

系统必须提供基于本地历史表的个股财务历史查询能力。

#### 场景：查看单只股票的历史财务

- **给定** 本地历史财务表中已经存在目标股票的历史记录
- **当** 用户查看该股票的财务历史时
- **那么** 系统从本地历史存储返回该股票的历史财务数据

### 需求：将 PDF 解析结果限定为人工辅助

系统必须将基于 PDF 的财务提取结果视为人工辅助，而不是正式筛选数据。

#### 场景：结构化财务数据暂时不可得

- **给定** 系统暂时无法获取结构化财务数据
- **当** 用户基于公告或年报 PDF 提取财务信息时
- **那么** 系统仅将提取结果作为人工辅助信息暴露出来
- **并且** 不自动写入正式财务快照
