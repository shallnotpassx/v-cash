# Design

## 目标

让公告中心独立存在，只依赖统一股票身份，不强依赖数据同步主链路。

## 核心决策

### 1. 独立边界

`announcement-center` 只依赖 `stock-registry`。

这样即使未来公告抓取和其它同步任务拆成不同入口，公告中心本身的查询和展示边界也不会混乱。

### 2. 存储原则

一期默认只保存：

- 股票引用
- 公告标题
- 公告类型
- 披露日期
- 来源站点
- 原文链接

默认不把 PDF 文件缓存到本地。

### 3. 人工解析入口

当结构化财务拿不到时，公告中心提供原文入口，供人工查看或后续挂接 PDF 解析流程。

但解析结果是否进入正式财务链路，由 `financial-screening` 单独控制。

## 风险与约束

- 公告站点页面结构变化可能影响抓取稳定性
- 不缓存原文时，后续解析依赖外部链接可用性
