# Design

## 目标

`stock-registry` 是整个项目的根能力域。

它只负责把股票本身管理清楚，不承担数据同步、财务分析或公告抓取的职责。

## 核心决策

### 1. 主标识

内部主标识使用 `market + stock_code`。

这样做的原因是：

- 能同时覆盖 A 股和港股
- 不依赖外部平台自己的主键
- 方便后续跨表关联

### 2. 最小字段集合

股票主数据至少需要包含：

- `market`
- `stock_code`
- `stock_name`
- `listed_status`
- `listing_date`
- `exchange`

其它扩展字段后续按 change 增量补充。

### 3. 边界

这个能力域只对外提供“股票是谁”和“股票基础资料是什么”。

它不负责：

- 自动回源补数据
- 财务指标计算
- 公告文档下载

## 数据模型方向

一期先按单表维护股票主数据。

如果未来需要支持别名、历史更名或多市场映射，再单独拆扩展表。

## 风险与约束

- 不同数据源对港股代码格式可能不一致，需要在接入层统一格式
- 股票退市、停牌等状态要保留，不能简单删除
