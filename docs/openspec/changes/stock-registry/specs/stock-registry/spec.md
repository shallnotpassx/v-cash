# 股票主数据规范

## 新增需求

### 需求：统一股票主标识

系统必须使用 `market + stock_code` 作为内部股票主标识。

#### 场景：为不同市场股票建立统一身份

- **给定** 系统需要同时维护 A 股和港股股票
- **当** 新建或更新一条股票主数据时
- **那么** 系统使用 `market + stock_code` 生成内部股票身份
- **并且** 不直接把用户原始输入作为主键或内部主标识

### 需求：维护独立的股票基础资料

系统必须将股票基础资料与财务、公告等其它能力域的数据分开维护。

#### 场景：保存最小股票画像

- **给定** 系统已经识别出一只股票
- **当** 保存该股票的主数据时
- **那么** 记录至少包含市场、股票代码、股票名称、上市状态、上市日期和交易所

### 需求：提供本地股票列表、搜索与详情查询

系统必须基于本地库存提供股票列表、基础搜索和详情查询能力。

#### 场景：按条件搜索本地股票

- **给定** 本地数据库已经存在股票主数据
- **当** 用户按市场、代码或名称搜索股票时
- **那么** 系统仅从本地存储返回匹配结果

#### 场景：查看单只股票详情

- **给定** 本地数据库中已经存在目标股票主数据
- **当** 用户按内部股票身份或市场加代码查询详情时
- **那么** 系统仅从本地存储返回该股票的基础资料
