# Design

## 目标

提供一个简单、可迁移的交付方式，让项目既能先在本地 Windows 机器启动，也能在后续迁移到 NAS 或 VPS 时尽量少改动。

## 核心决策

### 1. 交付结构

一期采用前后端分离的单体交付方式：

- 一个 Spring Boot 后端
- 一个 Vue 3 前端
- 一个 MySQL 实例

### 2. 启动方式

优先使用 Docker Compose 统一启动。

这样做的原因是：

- 本地搭建成本低
- 环境变量更集中
- 后续迁移到其它机器时，结构不需要推倒重来

### 3. 运行定位

前期按“本地优先”考虑，不把外网可访问当成前提条件。

当需要拉数据时，可以长期开机；平时则按需启动。

## 风险与约束

- 低配置机器上要尽量控制容器数量和镜像体积
- Windows 本地环境需要避免过度依赖复杂脚本
