# 部署规范

## 新增需求

### 需求：提供本地优先的交付结构

系统必须在当前项目阶段提供一套本地优先的交付结构。

#### 场景：为本地使用准备交付结构

- **给定** 项目仍处于规划和骨架阶段
- **当** 仓库准备交付与运行结构时
- **那么** 仓库中包含面向本地使用的交付布局

### 需求：统一启动核心服务

系统必须定义一种同时启动前端、后端和 MySQL 的方式。

#### 场景：启动本地运行栈

- **给定** 用户希望在本地启动整个项目
- **当** 使用约定的交付方式启动系统时
- **那么** 前端、后端和 MySQL 作为一套协调好的运行栈被一起启动

### 需求：通过环境变量管理运行配置

系统必须通过环境变量管理运行时配置项。

#### 场景：调整端口或数据库凭据

- **给定** 运行端口、数据库地址或凭据发生变化
- **当** 用户更新环境变量配置时
- **那么** 部署栈从环境变量中读取对应配置
